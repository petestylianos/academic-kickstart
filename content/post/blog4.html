Overview

In this blog-post I will review a new R package named covidexplorer authored by my colleague Jimmy Effendy. This package was developed as part of an assignment for unit ETC5523: Communicating with Data; a core unit in the master of Business Analytics at Monash University. The main purpose of the package is to launch a Shiny app that allows the user to explore data relating to COVID-19. More specifically, the Shiny app a provides users with insights of the spread of COVID-19 in the US through interactive plots and tables.

For more inforamtion you can refer to this link, or use the below browser.

Expectations vs. Reality

Expected

\(\color{green}\checkmark\) Installation

devtools::install_github("etc5523-2020/r-package-assessment-j-effendy")

Of to a good start. The package was installed quickly and without any errors or warnings.
Now let’s look at the exported datasets and functions.

\(\color{green}\checkmark\) Datasets

ALl datasets can be accessed and their relevant documentation offers explanations about their variables and it is very thorough and accurate.

For reference the package exports three datasets named:

  • covid: USA COVID-19 Data
  • state_list: USA State List.
  • usa_map: USA Map Data.

More info on how to use them can be found here

The final step is to check the exported functions,

\(\color{red}\bigotimes\) launch_app()

Unfortunately, I couldn’t get the app to launch as I was getting an error.

Reality

Error Produced when using covidexplorer::launch_app()

here() starts at /Library/Frameworks/R.framework/Versions/4.0/Resources/library/covidexplorer Warning in readChar(con, 5L, useBytes = TRUE) : cannot open compressed file /Library/Frameworks/R.framework/Versions/4.0/Resources/library/covidexplorer/data/covid.rda', probable reason 'No such file or directory' Error in readChar(con, 5L, useBytes = TRUE) : cannot open the connection

So, unfortunately I am not able to review the app inside the package.

However, I should mention that the error is not probably significant and it is due to a missing .rda object that I am sure will be fixed in a future update. Perhps the script that produces the shiny app needs to source this dataset at the beginning.

Suggestions

Other than resolving the issue with the launch_app() function, let’s see on what else can be further improved. The documentation of the package is excellent and details are provided for every function. I would suggest adding the link to the vignettes site next to the repository on GitHub in the About section. Unfortunately, I can’t provide any constructive feedback for the shiny app since I can’t launch it.

What I learned

In conclusion, it is evident that Jimmy put a lot of effort in developing this package and thoroughly documenting everything. Based on, what I see in the README the shiny app also provides many insights on the state of coronavirus in USA. What I learned is how prone R packages are to errors and how hard might be to debug them. I am guessing that the package worked perfectly for Jimmy at his local system, but a problem arises when somebody else downloads it from GitHub. The fact that I encouraged similar problems when I developed mine(without knowing if they are fixed at the moment) highlights the importance of developing packages in collaboration with others cause the burden of doing all the work at your own might be overwhelming, especially in the first stages of your development career.

Kudos to Jimmy Effendy for his hard work!! Can’t wait to see more of his next work.

Review Checklist

Documentation

The package includes all the following forms of documentation:

  • \(\color{green}\checkmark\) Installation instructions: for the package is found in README
  • \(\color{green}\checkmark\) Vignette(s) demonstrating major functionality that runs successfully locally
  • \(\color{green}\checkmark\) Function Documentation: for all exported functions in R help
  • Examples for all exported functions in R Help that run successfully locally

Functionality

  • \(\color{green}\checkmark\) Installation: Installation succeeds as documented.
  • \(\color{red}\bigotimes\) Functionality: Any functional claims of the software been confirmed.
  • [\(\color{green}\checkmark\)] Automated tests: Unit tests cover essential functions of the package
    and a reasonable range of inputs and conditions. All tests pass on the local machine.
